# v1.0 Acceptance Tests

The v1.0 candidate is acceptable only when these checks pass on the repository
without local secrets or external accounts.

## Required Commands

```powershell
python -m unittest discover -s tests
python -m compileall src tests
```

If frontend files changed:

```powershell
cd frontend
npm.cmd run build
```

For a release candidate or final v1.0 tag, run the frontend build even when no
frontend files changed.

## Release Smoke

- `POST /api/v2/live-agent-runs` starts the default Planner-led AgentGraph path.
- The live run reaches `completed` or a clearly explained Planner blocker.
- A completed live run returns `runtime_type=agent_graph` and `stored_run_id`.
- `GET /api/v2/runs/{stored_run_id}` returns the stored result.
- `GET /api/v2/runs/{stored_run_id}/events?cursor=0` returns stored events.
- At least one stored artifact can be read through
  `/api/v2/runs/{stored_run_id}/artifacts/{artifact_id}`.
- Old live-run detail/event endpoints are not product routes.

## Architecture Gates

- README, release plan, runtime action contract, deletion plan, and contract
  tests point to v1.0 readiness.
- Product live AgentGraph uses the frozen path:
  `AgentWorkflowSpec -> PlannerOrder.plan_graph -> RunController/RunGuard -> BudgetBroker round preflight -> GraphRunCache -> ActionGateway -> ContextService -> AgentRun -> PlannerStrategy -> AgentEngineRegistry -> Engines -> PlannerInputBundle -> PlannerDecision -> RunController`.
- Only Planner can ask the user. Non-Planner agents return artifact, blocker, or
  evidence records.
- Product live AgentGraph does not compile through the removed old workflow
  schema, run a fallback runner, or emit old runtime artifacts.
- Stored AgentGraph result read APIs remain available for events, artifacts,
  context packets, tool results, and blobs.
- Old workflow runtime endpoints are removed from the product API.

## Runtime Action Gates

- `repo_index` records repo intelligence through `ActionGateway`.
- `call_plugin` low-risk approved operations execute; high-risk or permissioned
  operations block with approval metadata.
- `call_mcp` defaults to approval before execution.
- `run_command_sandbox` accepts argv-based checks and stays inside the sandbox.
- Shell command execution outside sandbox requires approval.
- Unknown requested actions create failed runtime action artifacts rather than
  disappearing.
- Blocked plugin/MCP actions preserve approval key, policy, original
  `ActionSpec`, and `work_item_id`.
- Approved runtime action replay goes through `ActionGateway` without re-running
  executor model output generation.

## Coding Eval Gates

- Coding eval reports include `patch_preview`, `sandbox_apply`, `check_result`,
  `debug_finding`, and `runtime_action` counts/status.
- Failed check results and debug findings push Planner to continue/replan before
  finish.
- Successful second-round checks can finish without asking the user.

## Legacy Gates

- `/api/v2/agent-workflows/default` returns only the AgentWorkflow contract and
  does not include legacy `workflow`, `runtime_boundary`, `runtime_type`, or
  `deprecated` preview fields.
- Old compile/live-run workflow endpoints are not registered product routes.
- Old `plan_artifact`, `patch_artifact`, and `review_artifact` outputs are not
  emitted by product AgentGraph runs.
